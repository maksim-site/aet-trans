# Админка новостей

## Настройка

Создайте в корне проекта файл `.env.admin` по примеру `.env.admin.example`:

```text
AET_ADMIN_PASSWORD=надежный-пароль-не-короче-12-символов
AET_ADMIN_SESSION_SECRET=длинная-случайная-строка-для-подписи-сессий
```

Файл `.env.admin` добавлен в `.gitignore`, поэтому пароль не попадет в репозиторий.

## Запуск

Из корня проекта:

```bash
npm run admin
```

Адреса после запуска:

- сайт: `http://127.0.0.1:43119/`;
- админка: `http://127.0.0.1:43119/admin/`.

Без правильного пароля API новостей недоступен. Обычный вход действует до закрытия браузера и не более 8 часов. Галочка «Запомнить меня» сохраняет вход на 30 дней. В админке есть кнопка выхода.

## Возможности

- поиск и фильтр по году;
- создание, редактирование и удаление новостей;
- отдельное управление кратким описанием и его показом на странице;
- загрузка нескольких фотографий в JPG, PNG или WebP;
- выбор главной фотографии;
- автоматическая пересборка сайта;
- резервная копия данных перед каждым изменением;
- проверка внутренних ссылок после сохранения.

Исходные данные находятся в `v3/data/news.json`. Резервные копии создаются в `.admin-backups/`.

## Публикация

GitHub Pages размещает только статические файлы и не может запустить защищенный API админки. Для доступа Ольги админ-сервер нужно разместить на Node.js-хостинге с HTTPS и задать там `AET_ADMIN_PASSWORD`, `AET_ADMIN_SESSION_SECRET`, `NODE_ENV=production`.

По умолчанию сервер слушает только `127.0.0.1`. На сервере адрес задается через `AET_ADMIN_HOST`, например `0.0.0.0`. Публичный запуск без HTTPS недопустим.
